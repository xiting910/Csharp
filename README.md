## 项目说明

这个仓库中的项目是一些C#编程的尝试，旨在练习使用C#语言的各种功能和特性。主要项目包括：

- **GluttonousSnake**: 一个简单的贪吃蛇游戏实现，包含游戏逻辑、用户系统、数据存储等模块。
  - 使用了用户系统来管理程序，需要以玩家登录后才能进行游戏。
  - 输入密码时显示的是`*`，以保护隐私（这下自己都不知道、记不住密码了）。
  - 账号密码经过加密后存储在本地文件中，同时将该文件隐藏以保护隐私（虽然大多数时候只是防范到了自己）。
  - 目前发现程序运行时按`Esc`键有时会导致一些显示上的问题，暂时未找到原因，不过影响不大。
  - 运行程序后，若强行关闭窗口，数据文件不能正确设置隐藏属性，不清楚是否会造成BUG。
  - 游戏没有GUI界面，所有操作通过命令行进行，未来计划添加图形界面。
- **FabricServerManager**: 一个自动管理`Windows`系统的`Carpet`端`Fabric`服务器程序。
  - 设计这个程序主要是为了偷懒，不想每次都手动输入`stop`指令来关闭服务器。主要功能是在服务器无玩家活动一定时间后，程序会自动执行`stop`指令关闭服务器。
  - 支持在运行时修改程序的配置，如服务器无玩家活动后自动关闭的时间间隔、服务器启动的内存参数等。
  - 程序基于锂模组来识别服务器是否成功启动，可能不适用大部分`Fabric`服务器。
  - 可以通过修改`Constants.cs`文件中的`ServerStartSuccessMessage`变量来指定要识别的服务器启动成功消息。
  - 对于程序的识别指令执行功能，只有在装了`Carpet`模组的`Fabric`服务器并且服务器开启了`Carpet`的`recordPlayerCommand`选项时才能正常工作。
  - 程序支持上下键切换历史指令，并且可以通过回车键执行指令。
  - 输入行一直保持在末尾，不会被输出内容覆盖。
  - 未来计划添加`Esc`键退出功能和`Tab`键自动补全指令功能。
- **MineClearance**: 一个带有GUI界面的扫雷游戏实现，支持历史记录和多种难度。
  - 游戏界面使用了`WinForms`，提供了基本的扫雷游戏体验，不过界面较为单一。
  - 支持简单、中等、困难和自定义四种难度，其中自定义难度的游戏结果不参与排行榜，难度与https://www.minesweeper.cn/ 这个网站的难度相同（打开格子的数字颜色也与该网站基本一致）。
  - 布雷算法根据设置的雷的密度而有所不同，低密度时确保不会有雷的周围全部都是雷的情况，高密度时则完全随机布雷，但是都确保第一次点击不会踩到雷。
  - 由于是第一次点击后才布雷，所以第一次不能插旗。
  - 目前排行榜功能较为简单，游戏结果输出也较为基础，未来计划改进。
  - 暂时还未实现最后出现二选一的情况时的处理，所以最后几个雷可能要靠猜(众所周知，扫雷其实是一个运气游戏)。
  - 放弃了使用项目`GluttonousSnake`中的用户系统，毕竟只支持本地游玩。

## 系统要求

⚠️ **重要提醒**: 本仓库中的项目仅支持Windows操作系统运行。

## 运行环境
- **操作系统**: Windows 11(其他版本可能也可以，但未测试)
- **.NET版本**: .NET 8.0(项目中使用了很多现代的C#特性，建议使用最新版本)
- **IDE**: Visual Studio Code(其实可能Visual Studio更好，但我本人更喜欢VS Code)

## 兼容性说明
由于项目使用了Windows特定的API和功能，暂不支持Linux或macOS系统且难以修改适配。

## 下载和使用

### 前置要求
确保您的系统是Windows系统，并且已安装：
- .NET 8.0 SDK 或更高版本
- git 任意版本

### 克隆项目
```bash
git clone https://github.com/xiting910/Csharp.git
cd Csharp
```

### 运行项目
每个子项目都可以独立运行：
```bash
# 运行贪吃蛇游戏
cd GluttonousSnake
dotnet run

# 运行Fabric服务器管理器
cd FabricServerManager
dotnet run

# 运行扫雷游戏
cd MineClearance
dotnet run
```
